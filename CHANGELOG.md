# Changelog

本项目遵循语义化版本编号。

## 3.0.2 - 2026-07-01

### 隐私

- 从公开源码和发布制品中移除私人 Skill 与测试文件。
- 新安装使用空白技能库，私人 Skill 仅从本地导入。

## 3.0.1 - 2026-07-01

### 修复

- 已登记技能内容变化后会按哈希重新识别并体检。
- Bundle 额外项目文件默认禁止同步，必须经过独立授权。
- 集合同名成员会区分内容重复、上游更新和本地冲突。
- 普通导入与直接复制统一执行高风险及 AI 差异确认。
- 同步目标按 Windows 大小写无关规则检测路径冲突。
- 拒绝截断的 AI 差异，并保留自定义 frontmatter。
- 拒绝与技能库或暂存目录重叠的来源及 Junction/reparse point。

## 3.0.0 - 2026-06-29

### 新增

- 支持 Markdown、ZIP、标准 `SKILL.md` 文件夹和 `skills/*/SKILL.md` 集合导入。
- 无需 AI 即可完成元数据规范化、UTF-8 转换、安全扫描、去重和原版归档。
- 可选 AI 导入优化；接口不可用时自动回退本地结果。
- 自动发现直接复制到技能库的新文件，并提供原地体检。
- 集合以父卡形式展示，子技能可独立启用或停用。
- 安全同步预览、冲突检测、事务备份和最近一次同步撤销。
- API 配置状态与脱敏提示。
- 内置原版技能基线和首次启动本地适配。

### 修复

- 修复标准技能 YAML 多行简介显示为 `>` 的问题。
- 修复禁止记录敏感数据的规则被安全扫描误报的问题。
- 消除组合技能与独立技能的重复同步冲突。
- 防止 ZIP 路径穿越、符号链接导入和预览后文件替换。

### 变更

- 产品名称统一为 SkillHub。
- 活动技能库默认迁移到用户数据目录。
- 原版技能与运行时优化技能分离。
- PyInstaller 构建入口统一为 `SkillHub.spec`。

## 2.0.0 - 2026-06-25

- 增加 AI 对话、技能分类、自定义模型接口和本地资源打包。

## 1.0.0 - 2026-05-24

- 首个 Windows 免安装版本。
