# ⚡ 前端性能优化规范

本指南旨在为前端开发提供一套系统化的性能调优 checklist。在本地开发及生产打包时，请严格遵守以下规范。

## 📦 1. 资源打包优化
- **代码分割 (Code Splitting)**: 针对路由进行懒加载，对于第三方巨无霸库（如 Echards, lodash）使用 UMD/CDN 引入或单独分包。
- **Tree Shaking**: 确保使用 ES Modules 语法 (`import/export`)，避免引入无用代码。
- **体积压缩**: 开启 Gzip 或 Brotli 压缩，移除生产环境的 `console.log`。

## 🖼️ 2. 媒体与图片优化
- **现代格式**: 优先使用 WebP/AVIF 格式图片。
- **延迟加载 (Lazy Load)**: 对非首屏图片使用 `loading="lazy"` 或 IntersectionObserver 实现延迟加载。
- **大小适配**: 绝不允许将 4K 原图直接展示为 100x100 的头像，使用缩略图服务。

## 🚀 3. 运行时性能
- **减少重排与重绘**: 合理使用 CSS 属性 `will-change`，使用 `transform` / `opacity` 代替 `top` / `left` 做动画。
- **虚拟列表 (Virtual List)**: 当渲染超过 1000 条长列表数据时，必须使用虚拟滚动渲染，避免 DOM 节点过多导致卡顿。
- **防抖与节流**: 对于滚动、输入框实时搜索等高频触发事件，必须加防抖 (Debounce) 或节流 (Throttle)。
