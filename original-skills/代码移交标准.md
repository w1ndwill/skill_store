# 🤝 代码移交标准 (Handoff Checklist)

为了保障开发工作顺利交接，避免“人走代码死”的局面，请在准备移交代码或提交大版本 PR 时，对照本规范进行自查。

## 📖 1. 完善的文档支持
- **README.md**: 必须包含本地一键启动命令、环境变量依赖、生产打包命令以及项目整体架构简析。
- **API 接口文档**: 提供 Swagger 链接、Postman 导出的集合或在 README 中列出核心 API 接口说明。

## 🛠️ 2. 环境与依赖管理
- **依赖锁文件**: 必须提交 `package-lock.json`、`pnpm-lock.yaml` 或 `poetry.lock`，严禁本地随意升级大版本而不锁定。
- **环境变量模版**: 根目录下必须存在 `.env.example`，列出所有需要填写的敏感密钥和外部服务配置，并注明获取方式。

## 🧹 3. 代码整洁度与测试
- **无残留废弃代码**: 移交前清理所有注释掉的代码块、测试用的 `console.log` 以及未使用的依赖包。
- **Lint 与格式化**: 确保通过 ESLint / Prettier 检测，无 Error 级报错。
- **单元测试**: 核心业务逻辑必须有单元测试覆盖，且移交前测试全部通过。
