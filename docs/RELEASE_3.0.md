# SkillHub 3.0 发布说明

发布日期：2026-06-29

## 概要

SkillHub 3.0 将技能导入、检查、优化、集合管理和项目同步整合为一条可回滚的本地流程。AI 优化是可选能力；未配置 API 时，导入和同步仍可正常使用。

## 主要变化

### 技能导入

- 支持 `.md`、`.zip`、标准 `SKILL.md` 文件夹和技能集合。
- 导入前执行编码转换、元数据补齐、安全扫描和重复检测。
- 原始下载保存在活动技能库的 `.skill-hub/imports/upstream/`。
- AI 优化失败时保留本地处理结果，不阻塞安装。

### 集合管理

- 自动识别 `skills/*/SKILL.md` 结构。
- 集合在主界面显示为一张父卡。
- 子技能可独立启用或停用；停用不会删除源文件。
- 项目同步只装载集合中当前启用的子技能。

### 安全同步

- 写入项目前显示新增、修改、删除和冲突预览。
- 不覆盖项目中未经 SkillHub 管理的同名文件。
- 每次同步保存事务记录和备份，可撤销最近一次同步。
- 项目文件在预览后发生变化时拒绝继续写入。

### 原版与优化版分离

- `original-skills/` 保存随程序发布的原始基线。
- 活动技能库位于用户可写目录，默认路径为 `%LOCALAPPDATA%\SkillHub\skills`。
- 首次启动使用本地规则补齐元数据，并移除模板运行残留和重复项。

### Superpowers 工作流

Superpowers 作为一个组合技能提供四个子工作流：

1. `brainstorm`：需求分析、方案探索和风险识别。
2. `planning`：多文件或高风险任务的实施规划。
3. `tdd_execution`：以测试表达行为变化并驱动实现。
4. `verification`：回归检查、代码审查和交付确认。

## 安全边界

- SkillHub 不执行下载仓库中的插件 hooks、MCP 服务或安装脚本。
- API Key 只保存在本地 `config.json`，不会进入 Git 仓库或 Release 附件。
- GitHub Release 仅分发 `SkillHub.exe`。

## 验证

- 29 项自动化测试覆盖导入、集合、同步、撤销、会话和路径安全。
- Python 编译检查、JavaScript 语法检查和 PyInstaller 构建通过。

## 升级

直接使用新的 `SkillHub.exe` 替换旧版本即可。现有外部技能库和项目关联配置保持不变。
